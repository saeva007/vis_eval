## 2. 整体分类性能

### 2.1 分类报告与稀有类指标

测试集类别极不均衡：Fog 约 1.57 万、Mist 约 1.05 万、Clear 约 114 万。评估时**不以 Accuracy 为主**，而以 Fog/Mist 的 Recall、CSI、Precision 及低能见度整体 Precision、FPR 等为主。


| 类别    | Precision | Recall   | F1-score | Support   |
| ----- | --------- | -------- | -------- | --------- |
| Fog   | 0.25      | **0.49** | 0.33     | 15,682    |
| Mist  | 0.06      | 0.35     | 0.11     | 10,529    |
| Clear | 0.99      | 0.94     | 0.96     | 1,140,501 |


**稀有事件相关指标：**

- **Fog_CSI: 0.20**，Fog_POD (Recall): **0.49**，Fog_FAR: 0.75  
- **Mist_CSI: 0.06**，Mist_POD: 0.35，Mist_FAR: 0.94  
- **low_vis_precision: 0.19**，**false_positive_rate: 0.064**  
- **ECE: 0.015**，Brier（Fog/Mist）较低，概率校准尚可

**小结：** Fog 召回约一半，仍有大量漏报；Mist 精确率很低（6%）、虚警率高，是当前主要痛点之一；Clear 主导样本上整体准确率 92%，但低能见度端的 Precision/FPR 才是业务关键。

## 4. 混淆类型与瓶颈分析（Priority 1 诊断）

按 **bottleneck_table** 与分层混淆统计，全表错误中占比最高的混淆类型为：


| 混淆类型       | 全表 count | 全表 rate   | JJA 下 rate | 边界带 (400–1200 m) rate |
| ---------- | -------- | --------- | ---------- | --------------------- |
| clear→mist | 52,667   | **4.51%** | **5.23%**  | 9.51%                 |
| clear→fog  | 20,671   | **1.77%** | **2.18%**  | 3.97%                 |
| mist→clear | 4,704    | 0.40%     | 0.21%      | 26.85%                |
| fog→mist   | 4,052    | 0.35%     | 0.14%      | 3.76%                 |
| mist→fog   | 2,091    | 0.18%     | 0.12%      | 11.94%                |


**要点：**

1. **Clear→Mist / Clear→Fog** 是主要虚警来源，尤其在 **JJA（夏季）** 和 **白天**（confusion_summaries_stratified 中 day 的 clear→mist 占比更高），与“夏季对流/混合强、易把晴空判成轻雾”的物理一致。
2. **边界带（400–600 m、600–800 m、800–1200 m）** 内 mist→clear、mist→fog 等比例高，说明 500 m / 1000 m 附近仍是分类难点。
3. 后续可针对：**JJA + 白天** 的 Clear→Mist 做物理先验或后处理；对 **边界带** 做 Hard Negatives 或 pairwise margin 等损失设计（见项目 Priority 2–4）。

## 7. 结论与后续可讨论方向

### 7.1 当前表现小结

- **Fog**：Recall ≈ 49%，CSI ≈ 0.20；仍有约一半雾样本漏报，业务上需继续抬升召回。  
- **Mist**：Precision 仅约 6%，FAR 很高；Clear→Mist 虚警（尤其 JJA/白天）是主要瓶颈。  
- **边界带**：500 m、1000 m 附近 Fog↔Mist、Mist↔Clear 混淆集中，需联合优化而非单点调阈值。  
- **大范围事件**：PMST 相对 IFS 有明显优势，具备业务价值；事件内精度与时空分布可进一步细化分析。

### 7.2 建议与导师讨论的方向

1. **诊断与后处理（Priority 1–2）**
  - 是否在业务侧固定“Fog Recall 不低于当前”的约束下，对 JJA/白天做 **物理先验过滤**（如强混合、高 CAPE 时压低 Mist 预报），以降低 Clear→Mist。  
  - 是否对 **(fog_th, mist_th)** 或季节阈值做**联合扫描**，在保证 Fog_R 的前提下微调 Mist 精确率。
2. **数据与损失（Priority 3–4）**
  - 针对 **Fog 漏报** 与 **Clear→Mist 虚警** 构建 Hard Negatives / 边界带增强。  
  - 在 **DualBranchLoss** 中保留/加强 **pairwise margin（Fog vs Mist）** 与 **Clear margin**，强化 500 m 边界分离。
3. **情景与特征（Priority 5）**
  - 利用 **fig7 / fig8** 中季节、区域、沿海-内陆差异，讨论是否引入 **BLH/混合、低空剪切、暖季不稳定指数** 等 regime 特征，或分情景建模。
4. **架构（Priority 6，仅在 P1–P5 平台期后）**
  - 若上述手段收益饱和，再考虑 Regime-Conditioned Adapters 或 MoE 等结构，避免过早大改 backbone。
